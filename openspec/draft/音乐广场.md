WanaSing 广场功能设计文档
基于你的需求——记录用户操作、发布歌曲、引发互动、提高日活、给予灵感，我为你抽象设计一个完整的「广场」功能模块。

一、核心理念
维度	说明
功能名称	广场 / 灵感空间
核心价值	让用户的创作和练习被看见，激发他人行动
设计原则	低门槛发布、高密度灵感、强行动引导
目标指标	日活 +30%、创作量 +50%、互动率 +20%
二、用户行为抽象
广场记录的用户行为可分为 4 大类、12 种子类型：

2.1 发布类行为（产生内容）
行为类型	说明	展示形式	示例
发布演唱作品	用户完成的歌曲演唱	音频卡片 + 波形图	「张三 演唱了《十年》」
发布AI创作	哼唱生成的伴奏/歌曲	音频卡片 + 风格标签	「李四 AI 生成了一首钢琴曲」
发布练习片段	用户练习的精彩片段	短音频（15-30秒）	「王五 完成了气息练习」
发布动态/心情	文字+图片的创作心得	图文卡片	「终于唱上去了！🎉」
2.2 互动类行为（产生连接）
行为类型	说明	展示形式	示例
点赞/喜欢	对其他用户作品表示认可	通知 + 计数	「赵六 赞了你的作品」
评论/留言	对作品的文字反馈	评论列表	「唱得真好！求教程」
分享/转发	将作品分享到站外/站内	分享记录	「小红 分享了这首歌」
收藏/保存	收藏作为学习素材	收藏记录	「小明 收藏了这首歌」
2.3 学习类行为（产生参考价值）
行为类型	说明	展示形式	示例
跟唱/模仿	用户演唱同一首歌	关联展示	「3人跟唱了这首歌」
完成挑战	参与官方/用户发起的挑战	挑战榜单	「小美 完成了#高音挑战#」
获得徽章	达成某项成就	徽章卡片	「小强 获得了「气息大师」徽章」
2.4 数据类行为（产生激励）
行为类型	说明	展示形式	示例
评分提升	演唱分数进步	进步卡片	「你的音准提升了15分！」
完成目标	完成每日练习目标	成就展示	「今日练习目标已达成」
连续打卡	连续多日使用	打卡记录	「连续练习7天 🔥」

6.3 算法推荐策略
python
def calculate_feed_score(feed):
"""计算 Feed 的推荐分数"""
# 基础分：互动率（点赞+评论+分享+播放）
engagement = (feed.likes + feed.comments + feed.shares) / feed.impressions

    # 时效分：越新越高
    hours_passed = (now - feed.created_at).hours
    recency = math.exp(-hours_passed / 24)  # 24小时衰减
    
    # 质量分：评分、完成度等
    quality = feed.score / 100 if feed.score else 0.5
    
    # 社交分：是否来自关注用户
    social = 1.5 if feed.user.is_followed else 1.0
    
    # 新鲜度：避免重复推荐
    diversity = 1.0 / (1 + feed.user.seen_count)
    
    return engagement * recency * quality * social * diversity
七、数据模型设计
7.1 Feed 表
sql
CREATE TABLE feeds (
id UUID PRIMARY KEY,
user_id UUID NOT NULL,
type VARCHAR(20) NOT NULL,  -- performance/creation/practice/like/comment/achievement
content JSONB NOT NULL,
metrics JSONB DEFAULT '{"likes":0,"comments":0,"shares":0,"plays":0}',
status VARCHAR(20) DEFAULT 'active',  -- active/deleted/reported
created_at TIMESTAMP DEFAULT NOW(),
updated_at TIMESTAMP
);

CREATE INDEX idx_feeds_created_at ON feeds(created_at DESC);
CREATE INDEX idx_feeds_user_id ON feeds(user_id);
CREATE INDEX idx_feeds_type ON feeds(type);
7.2 用户互动表
sql
CREATE TABLE user_actions (
id UUID PRIMARY KEY,
user_id UUID NOT NULL,
feed_id UUID NOT NULL,
action_type VARCHAR(20),  -- like/comment/share/save/follow
comment_text TEXT,
created_at TIMESTAMP DEFAULT NOW(),
UNIQUE(user_id, feed_id, action_type)
);
7.3 挑战/话题表
sql
CREATE TABLE challenges (
id UUID PRIMARY KEY,
title VARCHAR(100),
description TEXT,
song_id VARCHAR(100),
start_date DATE,
end_date DATE,
participant_count INT DEFAULT 0,
reward JSONB,
created_at TIMESTAMP DEFAULT NOW()
);
