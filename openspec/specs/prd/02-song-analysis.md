| 属性 | 内容 |
|------|------|
| 产品名称 | 歌曲多维度分析 |
| 版本号 | V1.0 MVP |
| 文档状态 | 定稿 |
| 目标平台 | H5 / Web |

## 1. 项目背景

### 1.1 市场痛点

| 痛点 | 说明 |
|------|------|
| 选歌难 | 不知道哪首歌适合自己的音域 |
| 学歌慢 | 不知道歌曲难度，练歌效率低 |
| 无反馈 | 练习时不知道难点在哪里 |
| 信息缺 | 想唱一首歌，不知道调性、音域等信息 |

### 1.2 产品定位

歌曲分析 是一款 AI 驱动的歌曲分析工具，上传一首歌曲，自动分析调性、音域、节奏、难度等，生成可视化报告和演唱建议。

### 1.3 核心价值

| 价值 | 说明 |
|------|------|
| 📊 多维度分析 | 音域、节奏、调性、结构全解析 |
| 🎯 难度评估 | 1-10 级难度分级，一目了然 |
| 💡 演唱建议 | AI 生成针对性的演唱技巧提示 |
| 🎵 音域匹配 | 对比用户音域，推荐合适的歌曲 |

## 2. 功能需求

### 2.1 功能总览

```
歌曲分析 V1.0
├── P0 核心功能（必须有）
│   ├── 音频上传
│   ├── 音高分析
│   ├── 节奏分析
│   ├── 结构分析
│   ├── 难度评分
│   └── 报告展示
├── P1 重要功能（应有）
│   ├── 历史记录
│   ├── 分析进度
│   └── 分享报告
└── P2 增强功能（可以有）
    ├── 用户音域管理
    ├── 歌曲推荐
    └── 歌词同步
```

### 2.2 P0 核心功能详解

#### F-01: 音频上传

| 项目 | 说明 |
|------|------|
| 功能描述 | 用户上传音频文件进行分析 |
| 支持格式 | MP3, WAV, M4A |
| 文件大小 | ≤ 50MB |
| 交互 | 拖拽上传 + 文件选择 |

#### F-02: 音高分析

| 项目 | 说明 |
|------|------|
| 分析内容 | 最低音、最高音、音域宽度、平均音高、音高变化 |
| 输出格式 | MIDI 编号 + 音符名称 + 可视化波形 |
| 技术方案 | Librosa + PYIN 算法 |

#### F-03: 节奏分析

| 项目 | 说明 |
|------|------|
| 分析内容 | BPM、节拍数、歌曲时长、切分音密度、节奏类型 |
| 节奏类型 | 慢板 (BPM<80)、中板 (80-120)、快板 (>120) |
| 输出格式 | 数值 + 分类标签 |

#### F-04: 结构分析

| 项目 | 说明 |
|------|------|
| 分析内容 | 段落边界检测、段落分类（主歌/副歌/前奏/尾声） |
| 输出格式 | 时间戳 + 段落类型 + 可视化时间轴 |

#### F-05: 难度评分

| 项目 | 说明 |
|------|------|
| 评分维度 | 音域宽度 (30%)、最高音 (20%)、BPM (15%)、其他 (35%) |
| 难度等级 | 1-3级（初学者）、4-6级（进阶）、7-10级（专业） |
| 输出格式 | 总分 + 各维度得分 + 等级标签 |

#### F-06: 报告展示

| 项目 | 说明 |
|------|------|
| 展示内容 | 所有分析结果 + 可视化图表 + 演唱建议 |
| 演唱建议 | 根据分析结果生成针对性提示 |
| 交互 | 可播放原曲对照 |

### 2.3 P1 重要功能详解

#### F-07: 历史记录

| 项目 | 说明 |
|------|------|
| 功能描述 | 保存用户的分析历史 |
| 存储 | 数据库持久化 |
| 展示 | 列表显示歌曲名 + 难度 + 分析时间 |

#### F-08: 分析进度

| 项目 | 说明 |
|------|------|
| 功能描述 | 分析过程中显示进度和状态 |
| 反馈 | 显示"正在分析音高..."、"正在检测节奏..." |

#### F-09: 分享报告

| 项目 | 说明 |
|------|------|
| 功能描述 | 分享分析报告给他人 |
| 方式 | Web Share API + 复制链接 |

## 3. 分析维度详情

### 3.1 音高分析输出

```json
{
  "lowest_note": 48,
  "lowest_note_name": "C3",
  "highest_note": 72,
  "highest_note_name": "C5",
  "vocal_range": 24,
  "mean_pitch": 60,
  "pitch_variation": 8.5,
  "max_jump": 12
}
```

### 3.2 节奏分析输出

```json
{
  "bpm": 120,
  "beat_count": 480,
  "duration_seconds": 240,
  "syncopation_score": 3.5,
  "rhythm_type": "medium"
}
```

### 3.3 难度评分输出

```json
{
  "total": 6,
  "level": "intermediate",
  "level_name": "进阶",
  "breakdown": {
    "vocal_range": 6,
    "highest_note": 7,
    "tempo": 5
  }
}
```

## 4. 演唱建议生成规则

| 条件 | 建议 |
|------|------|
| 音域 > 12 semitones | "这首歌音域较宽，注意换声区的平滑过渡" |
| 最高音 > C5 (MIDI 72) | "副歌有较高音区，建议用混声/假声演唱" |
| 切分音 > 2 | "切分音较多，注意节奏的准确度" |
| BPM > 140 | "速度较快，注意气息控制" |

## 5. 技术要求

### 5.1 性能指标

| 指标 | 目标 |
|------|------|
| 分析时长 | ≤ 30 秒（3 分钟歌曲） |
| 并发支持 | 50 QPS |
| 缓存命中 | > 60%（相同歌曲） |

### 5.2 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Python + FastAPI |
| 音频处理 | Librosa |
| 音高检测 | PYIN 算法 |
| 数据库 | SQLite / MySQL |
