# 哼唱成曲 - 产品需求文档

## 文档属性

| 属性 | 内容 |
|------|------|
| 产品名称 | 哼唱成曲 (Hum to Track) |
| 版本号 | V1.0 MVP |
| 文档状态 | 定稿 |
| 目标平台 | H5 / Web |

## 1. 项目背景

### 1.1 市场痛点

| 痛点 | 说明 |
|------|------|
| 创作门槛高 | 普通人脑海中有旋律，但不懂乐理、不会乐器、无法编曲 |
| 版权风险大 | 使用现有歌曲伴奏存在侵权风险 |
| 工具太复杂 | 专业 DAW 软件学习成本高，不适合普通人 |
| 反馈缺失 | 哼唱后不知道效果如何，无法改进 |

### 1.2 产品定位

哼唱成曲 是一款极简的 AI 音乐生成工具。用户只需哼唱 8-15 秒旋律，AI 自动生成完整伴奏，支持多种风格，一键下载分享。

### 1.3 核心价值

| 价值 | 说明 |
|------|------|
| 🎤 零门槛 | 会哼唱就能创作，无需乐理知识 |
| ⚡ 极速生成 | 30 秒内完成，即哼即用 |
| 🎨 风格多样 | 支持流行、钢琴、Lo-fi、摇滚等 6+ 风格 |
| 📤 一键分享 | 生成链接/下载 MP3，随时分享 |
| 💰 低成本 | 免费使用，API 按量付费 |

## 2. 用户画像

### 2.1 目标用户

| 用户类型 | 特征 | 使用场景 |
|----------|------|----------|
| 🎵 音乐爱好者 | 喜欢唱歌、有创作欲，不懂编曲 | 记录灵感、制作伴奏 |
| 📱 普通用户 | 觉得有趣、想尝试 AI | 娱乐、分享朋友圈 |
| 🎬 内容创作者 | 需要背景音乐 | 短视频、Vlog 配乐 |
| 👶 初学者 | 刚开始学音乐 | 练习旋律创作 |

### 2.2 用户故事

**小明（音乐爱好者）**
> 是个喜欢唱歌的大学生，经常有旋律在脑海里浮现，但不会编曲。他打开「哼唱成曲」，哼了 10 秒，选择「钢琴曲」风格，30 秒后得到一首完整的钢琴伴奏。他下载下来，配上自己的歌词，发到了朋友圈。

**小红（内容创作者）**
> 是个 Vlog 博主，需要为视频配背景音乐。她用「哼唱成曲」哼了一段轻快的旋律，选择「Lo-fi」风格，生成的音乐恰好符合视频氛围，再也不用担心版权问题。

## 3. 功能需求

### 3.1 功能总览

```
哼唱成曲 V1.0
├── P0 核心功能（必须有）
│   ├── 录音功能
│   ├── 音频可视化
│   ├── 风格选择
│   ├── AI 生成伴奏
│   └── 播放/下载
├── P1 重要功能（应有）
│   ├── 历史记录
│   ├── 分享功能
│   ├── 加载动画
│   └── 错误提示
└── P2 增强功能（可以有）
    ├── 歌词生成
    ├── 人声分离
    └── 社交分享卡片
```

### 3.2 P0 核心功能详解

#### F-01: 录音功能

| 项目 | 说明 |
|------|------|
| 功能描述 | 用户点击按钮开始/停止哼唱录音 |
| 输入 | 麦克风音频流 |
| 输出 | MP3 音频 Blob |
| 约束 | 最少 3 秒，建议 8-15 秒 |

**交互流程：**
1. 首次点击 → 请求麦克风权限
2. 权限通过 → 开始录音，按钮变红色+脉冲动画
3. 显示计时器（秒数）
4. 再次点击 → 停止录音
5. 录音时长 < 3 秒 → 提示"太短了，再哼一会儿吧"

#### F-02: 音频可视化

| 项目 | 说明 |
|------|------|
| 功能描述 | 录音时实时显示音量波形 |
| 技术 | Web Audio API + Canvas |
| 反馈 | 音量越大，波形条越高 |

#### F-03: 风格选择

| 项目 | 说明 |
|------|------|
| 功能描述 | 用户选择生成的音乐风格 |
| 选项 | 流行、钢琴曲、Lo-fi、摇滚、R&B、电子 |
| 默认 | 流行 |
| 交互 | 点击选中，高亮显示 |

#### F-04: AI 生成伴奏

| 项目 | 说明 |
|------|------|
| 功能描述 | 将哼唱音频发送给 AI，返回伴奏 |
| 输入 | 哼唱音频 + 风格标签 |
| 输出 | 30 秒伴奏音频 URL |
| 耗时 | 15-30 秒 |
| 技术 | 调用 Suno API / APIPASS |

**API 调用流程：**
```
用户哼唱 → 录音 Blob → Base64 编码 → 调用 API → 返回音频 URL → 前端播放
```

#### F-05: 播放与下载

| 项目 | 说明 |
|------|------|
| 功能描述 | 用户可试听生成的伴奏，并下载 |
| 播放 | HTML5 Audio 控件 |
| 下载 | 文件名为 wanasing_时间戳.mp3 |

### 3.3 P1 重要功能详解

#### F-06: 历史记录

| 项目 | 说明 |
|------|------|
| 功能描述 | 保存最近 10 次生成的伴奏 |
| 存储 | localStorage / 数据库 |
| 展示 | 列表显示风格 + 日期，点击可播放 |

#### F-07: 分享功能

| 项目 | 说明 |
|------|------|
| 功能描述 | 用户可将伴奏分享给他人 |
| 方式 | Web Share API（原生分享）/ 复制链接 |
| 降级 | 不支持时复制链接 |

#### F-08: 加载动画

| 项目 | 说明 |
|------|------|
| 功能描述 | AI 生成过程中展示等待动画 |
| 形式 | 旋转 Spinner + 趣味文案 |
| 文案示例 | "正在召唤吉他手..."、"鼓手已就位..." |

#### F-09: 错误提示

| 场景 | 提示文案 |
|------|----------|
| 麦克风无权限 | "无法访问麦克风，请在设置中开启" |
| 哼唱时间太短 | "哼唱时间太短，请至少哼唱 3 秒" |
| API 调用失败 | "生成失败，请稍后重试" |
| 网络异常 | "网络连接异常，请检查网络" |

## 5. 用户体验流程

### 5.1 主流程

```
进入页面 → 选择风格 → 点击录音 → 哼唱 8-15 秒 → 停止录音 → 点击生成 → 等待 15-30 秒 → 播放/下载/分享
```

### 5.2 异常流程

```
进入页面 → 麦克风权限拒绝 → 提示引导 → 用户手动开启 → 重新加载
```

### 5.3 状态流转

```
初始状态 → 录音中 → 已录音 → 生成中 → 已完成
  ↓          ↓         ↓        ↓
（可切换风格）（显示计时）（启用生成）（播放/下载）
```

## 7. 附录

### 7.1 竞品对比

| 产品 | 优势 | 劣势 |
|------|------|------|
| Suno | 质量高、风格多 | 需要文字描述，不支持哼唱 |
| Udio | 音质好 | 同上 |
| 网易天音 | 中文优化 | 功能复杂 |
| 哼唱成曲 | 哼唱输入、极简、免费 | 初期功能少 |

### 7.2 风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| API 不稳定 | 中 | 多备几个 API 服务商 |
| 浏览器兼容 | 低 | 提供降级提示 |
| 用户留存低 | 高 | 快速迭代，增加趣味功能 |
| 版权问题 | 低 | 用户生成内容归用户 |

### 7.3 成功指标

| 指标 | 目标值 |
|------|--------|
| 日活用户 | 100+（上线 1 个月） |
| 完成率（录音→生成） | > 60% |
| 分享率 | > 15% |
| 用户满意度 | > 4.0/5.0 |
